# Ground-truth 纪律

一条断言只有在核过对应产物后才成为证据。主 Agent 必须验证承重判定，不必重复 Worker 的所有低风险探索。

## 常见失真

| 失真 | 风险 | 核验动作 |
|---|---|---|
| 回执等同产物 | 命令接受了请求，但实体未落盘或字段丢失 | 写后读取目标实体、文件与版本记录 |
| diff 等同全文 | diff 看不到重复、残留和文件整体不变量 | 对关键不变量运行全文 `rg` 或结构化检查 |
| 分页等同全集 | 默认窗口遗漏旧记录 | 使用全量、搜索或聚合入口，并确认分页语义 |
| 快照等同终态 | Agent 或进程仍运行时读到中途状态 | 确认执行线结束后再判产物 |
| 沉默等同不存在 | 检测器可能未执行、路径错误或规则陈旧 | 用已知应命中的输入做阳性对照 |
| 记忆等同真实 gate | preset、profile、CI 和治理规则会演进 | 从目标仓权威源派生当前门集 |
| 单次测量等同趋势 | 噪声或调度被误判为改进/回归 | 取独立样本并比较样本间方差 |

## 两种对照

- **阳性对照**：让检测器对已知应命中的输入出声，获得信任其“无发现”的资格。
- **阴性对照**：让同一复现在改动前或权威基线上通过，获得归因给本改动的资格。两边都失败，只能说明复现没有分辨力。

## 落到编排

- Agent 的枚举可作为搜索起点；承重判定由主 Agent 回读 ground-truth。
- `ha` 写入后用 show/list 或目标文件回读；不要只保存 JSON 回执。
- 测试结论写明命令、环境、退出状态和关键输出；没运行的标 `未验证`。
- 语义审计从原始 decision/需求逐条走向实现，不用记忆决定检查范围。
- 汇报前写清每个结论依赖的仪器，以及该仪器如何被验证。
